# DateField
A form input component for entering and editing date and time values. The DateField component provides a user-friendly interface for date selection with support for various granularities (day, hour, minute, second) and customizable formatting options.

## Example usage
```tsx
<DateField
  label="Select Date"
  defaultValue={parseDate('2020-01-23')}
  granularity="day"
  description="Format: d MMM yyyy"
/>
```

## Sizes

* **small** - Compact size with smaller text and no calendar icon, ideal for dense layouts
* **medium** - Standard size with calendar icon and larger text, suitable for most use cases

**medium** is the default size and should be used in most cases, unless space constraints require the smaller variant.

## Props

### Core Props
- **`label`** - Text label displayed above the input field
- **`description`** - Helper text displayed below the input field
- **`errorMessage`** - Error message displayed when validation fails
- **`placeholder`** - Placeholder text shown when no value is entered
- **`isRequired`** - Whether the field is required (adds required indicator to label)
- **`isDisabled`** - Whether the field is disabled
- **`isInvalid`** - Whether to show the field in an invalid state

### Size & Appearance
- **`size`** - Size variant: `'small'` | `'medium'` (default: `'medium'`)
- **`shortMonth`** - Whether to display month names in short format (e.g., "JAN" vs "01") (default: `true`)

### Date Configuration
- **`granularity`** - Date precision: `'day'` | `'hour'` | `'minute'` | `'second'` (default: `'day'`)
- **`defaultValue`** - Initial date value
- **`value`** - Controlled date value
- **`onChange`** - Callback when date value changes

### Styling
- **`classNames`** - Object containing custom class names for different parts:
  - `field` - Main container
  - `label` - Label element
  - `input` - Input container
  - `description` - Description text
  - `error` - Error message

### Advanced Props
- **`slot`** - Slot name for the field
- **`name`** - Form field name
- **`id`** - Unique identifier for the field

## Implementation Notes
The `DateField` component is built on top of React Aria Components, specifically wrapping the `DateField` and `DateInput` components.

[Docs](https://react-spectrum.adobe.com/react-aria/DateField.html)

[Github Source](https://github.com/gohypergiant/standard-toolkit/tree/main/design-toolkit/components/src/components/date-field)

Key features:
- **Internationalization**: Uses `@internationalized/date` for robust date handling across locales
- **Accessibility**: Full keyboard navigation and screen reader support
- **Validation**: Built-in error state handling and ARIA attributes
- **Flexible Formatting**: Supports various date granularities and custom month formatting
- **Responsive Design**: Adapts to different screen sizes with size variants

The underlying HTML structure uses semantic form elements with proper ARIA labeling and error handling.
