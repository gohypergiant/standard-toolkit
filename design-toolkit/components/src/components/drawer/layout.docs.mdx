# DrawerLayout

A flexible layout system for managing multiple drawers that can be positioned on any side of the main content area. Supports pushing and extending behaviors with configurable sizes and placements.

<Canvas of={DrawerLayoutStories.Default} />

## Structure

The DrawerLayout system consists of the following components:

```tsx
<DrawerLayout>
  <DrawerLayoutMain>
    {/* Main content area */}
  </DrawerLayoutMain>

  <Drawer>
    <DrawerMenu>
      <DrawerMenuItem />
    </DrawerMenu>
    <DrawerPanel>
      <DrawerHeader>
        <DrawerHeaderTitle />
      </DrawerHeader>
      <DrawerContent>
        <DrawerView />
      </DrawerContent>
    </DrawerPanel>
  </Drawer>
</DrawerLayout>
```

- **DrawerLayout**: Container that manages drawer interactions and layout
- **DrawerLayoutMain**: Main content area that responds to drawer states
- **Drawer**: Individual drawer container with configurable placement
- **DrawerMenu**: Navigation menu for controlling drawer views
- **DrawerMenuItem**: Interactive item for toggling drawer states
- **DrawerPanel**: Content container for the drawer
- **DrawerHeader**: Header section of the drawer
- **DrawerHeaderTitle**: Title component for the drawer
- **DrawerContent**: Main content area within the drawer
- **DrawerView**: Individual view container within the drawer

## Props

### DrawerLayout Props

- `extend`
  - Type: select ('top bottom', 'left right', 'top', 'bottom', 'left', 'right')
  - Default: ''
  - Description: Specifies which drawers should extend over the main content

- `push`
  - Type: string
  - Default: ''
  - Description: Specifies which drawers should push the main content

- `children`
  - Type: ReactNode
  - Required
  - Description: Content to render within the layout

### Drawer Props

- `id`
  - Type: string
  - Required
  - Description: Unique identifier for the drawer

- `placement`
  - Type: select ('top', 'bottom', 'left', 'right')
  - Required
  - Description: Position of the drawer

- `size`
  - Type: select ('medium')
  - Default: 'medium'
  - Description: Size of the drawer

### DrawerMenuItem Props

- `toggle`
  - Type: boolean
  - Default: false
  - Description: Whether this item toggles the drawer state

- `for`
  - Type: string
  - Required
  - Description: ID of the view this item controls

### DrawerView Props

- `id`
  - Type: string
  - Required
  - Description: Unique identifier for the view

## Implementation Notes

- Built with TailwindCSS for styling
- Uses CSS Grid for layout management
- Handles transitions and animations automatically
- **Props use camelCase** (e.g., `placement`, `size`)
- **Component names use PascalCase** (e.g., `DrawerLayout`)
- **Descriptions begin with uppercase letters**

## Usage Requirements

- Must include exactly one `DrawerLayoutMain` component
- Each `Drawer` must have a unique `id`
- Each `DrawerView` must have a unique `id`
- `placement` must be one of: 'top', 'bottom', 'left', 'right'
- Menu items must reference valid view IDs

## Accessibility

- Keyboard navigation support for drawer controls
- ARIA attributes for drawer states and controls
- Focus management between drawers
- Support for reduced motion preferences
- Screen reader announcements for drawer state changes

## Contextual Use Cases

- Application navigation systems
- Tool panels and sidebars
- Property inspectors
- Configuration panels
- Multi-panel dashboards
- Complex form layouts
- Responsive mobile navigation
