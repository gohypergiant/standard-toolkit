## 5. Ecosystem Guidelines

### Component Aliases

When users request components by names from other UI frameworks, use these Design Toolkit equivalents:

| Common Name / Alias | Design Toolkit Component |
|---------------------|--------------------------|
| `CUI`, `Classification`, `ClassificationMarkings` | **ClassificationBanner** or **ClassificationBadge** |
| `Alert`, `Message`, `Callout`, `InlineAlert`, `Snackbar`, `Notification`, `Toast` | **Notice** and **NoticeList** |
| `Modal`, `Popup`, `Overlay` | **Dialog** |
| `BottomSheet`, `Side Panel`, `ActionSheet` | **Drawer** |
| `Placeholder`, `Shimmer` | **Skeleton** |
| `Menu`, `ContextMenu`, `DropdownMenu` | **Menu** |
| `Popup`, `HoverCard` | **Popover** |
| `Input`, `TextInput`, `FormField` | **TextField** |
| `Dropdown`, `Picker`, `Select` | **SelectField** |
| `Toggle` | **Switch** |
| `Search` | **SearchField** |
| `Calendar`, `DateInput`, `TimeInput`, `DatePicker`, `TimePicker` | **DateField** or **TimeField** |
| `Range`, `RangeSlider` | **Slider** |
| `ChipInput`, `MultiSelect`, `TagsInput`, `Combobox` | **ComboBoxField** |
| `FAB`, `FloatingActionButton` | **Button** |
| `Baseball Card`, `Object Details` | **Drawer** + **Hero** + **DetailsList** |
| `Coordinates Input`, `Coordinates`, `MRGS`, `GARS` | **CoordinateField** |

### Recommended Third-Party Libraries

Use these when functionality doesn't exist in `@accelint/**` packages:

- **[radashi](https://github.com/radashi-org/radashi)**: Low-level utilities
- **[@turfjs/**](https://github.com/Turfjs/turf)**: Geospatial functions

### Documentation Access (If Available)

If your environment supports MCP (Model Context Protocol) servers:
- Use [Context7 MCP](https://context7.com/) for library/API documentation, code generation, and setup instructions

If MCP is not available, search web documentation or ask user for guidance on library-specific patterns.
