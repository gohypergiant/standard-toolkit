# Hero
A versatile hero component that displays an icon alongside primary and secondary content. Supports both stacked and grid layouts.

Components:
- **Hero** - Container with automatic layout (renders as `<header>`)
- **HeroTitle** - Primary heading (renders as `<h2>`)
- **HeroSubtitle** - Secondary text content

## Example Usage

```tsx
<Hero>
  <Icon><Placeholder /></Icon>
  <HeroTitle>Primary Title</HeroTitle>
  <HeroSubtitle>Secondary information</HeroSubtitle>
</Hero>
```
<Canvas of={HeroStories.Default} />

### Compact Layout
```tsx
<Hero compact>
  <Icon><Settings /></Icon>
  <HeroTitle>Settings</HeroTitle>
  <HeroSubtitle>Configure your preferences</HeroSubtitle>
</Hero>
```

### Multiple Subtitles
```tsx
<Hero>
  <Icon><User /></Icon>
  <HeroTitle>John Doe</HeroTitle>
  <HeroSubtitle>Software Engineer</HeroSubtitle>
  <HeroSubtitle>San Francisco, CA</HeroSubtitle>
</Hero>
```

## Layouts

| Layout | Prop | Description |
|--------|------|-------------|
| Stack | `compact={false}` | Vertical layout with larger icon above stacked content (default) |
| Grid | `compact={true}` | Horizontal layout with smaller icon beside content |

## Props

### Hero
Extends all standard HTML `<header>` attributes.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `compact` | `boolean` | `false` | Use grid layout instead of stack |
| `classNames` | `object` | - | Custom classes (see below) |

**classNames:**

| Key | Description |
|-----|-------------|
| `hero` | Main container (`<header>` element) |
| `icon` | Icon container |
| `title` | Title text |
| `subtitle` | Subtitle text |

### HeroTitle
Renders as an `<h2>` heading element. Exactly one required per Hero.

### HeroSubtitle
Secondary text content. Zero or more allowed per Hero.

## Accessibility
- Uses semantic `<header>` element with proper heading hierarchy
- HeroTitle renders as `<h2>` for screen reader navigation

## Implementation Notes
- Icon is automatically sized based on layout mode
- Uses React Aria's HeadingContext and TextContext for styling
