# Table

Configurable data table with sorting, selection, and row actions built on TanStack React Table.

```tsx
import { createColumnHelper } from '@tanstack/react-table';

type Person = {
  id: string;
  firstName: string;
  lastName: string;
};

const columnHelper = createColumnHelper<Person>();

const columns = [
  columnHelper.accessor('firstName', {
    id: 'firstName',
    cell: (info) => info.getValue(),
    header: () => <span>First Name</span>,
  }),
  columnHelper.accessor('lastName', {
    id: 'lastName',
    cell: (info) => <i>{info.getValue()}</i>,
    header: () => <span>Last Name</span>,
  }),
];

const data = [
  { id: 'tanner', firstName: 'Tanner', lastName: 'Linsley' },
  { id: 'joe', firstName: 'Joe', lastName: 'Dirte' },
];

<Table columns={columns} data={data} showCheckbox />
```

## Static Table

For simple tables without sorting or selection, use subcomponents directly.

```tsx
<Table>
  <TableHeader>
    <TableRow>
      <TableHeaderCell>First Name</TableHeaderCell>
      <TableHeaderCell>Last Name</TableHeaderCell>
    </TableRow>
  </TableHeader>
  <TableBody>
    {data.map((person) => (
      <TableRow key={person.id}>
        <TableCell>{person.firstName}</TableCell>
        <TableCell>{person.lastName}</TableCell>
      </TableRow>
    ))}
  </TableBody>
</Table>
```

## Column Sizing

Define a `size` property in column definitions to set fixed widths.

```tsx
columnHelper.accessor('firstName', {
  id: 'firstName',
  header: 'First Name',
  size: 150, // Width in pixels
})
```

## Row Selection

Track selected rows with `rowSelection` and `onRowSelectionChange`.

```tsx
const [selectedRows, setSelectedRows] = useState({ tanner: true, joe: true });

<Table
  columns={columns}
  data={data}
  showCheckbox
  rowSelection={selectedRows}
  onRowSelectionChange={setSelectedRows}
/>
```

## Pagination

Table handles client-side data slicing internally when `pageSize` is set. Render `<Pagination>` alongside the Table to provide page navigation — the two components are kept separate so you have full control over Pagination's placement and props.

```tsx
const [page, setPage] = useState(1);

<Table columns={columns} data={data} pageSize={10} page={page} onPageChange={setPage} />
<Pagination value={page} total={Math.ceil(data.length / 10)} onChange={setPage} />
```

### Server-side Pagination

For server-side (pre-paginated) data, manage page state externally and pass only the current page's data. Use `key={page}` on the Table to force a remount when the page changes.

```tsx
const [page, setPage] = useState(1);
const pageData = allData.slice((page - 1) * PAGE_SIZE, page * PAGE_SIZE);

<Table key={page} columns={columns} data={pageData} />
<Pagination value={page} total={totalPages} onChange={setPage} />
```

### Static Table Subcomponents

For simple tables without sorting or selection, use these subcomponents directly:

| Component | Description |
|-----------|-------------|
| `TableHeader` | Renders `<thead>` section |
| `TableHeaderCell` | Individual header cell `<th>` |
| `TableBody` | Renders `<tbody>` section |
| `TableRow` | Table row `<tr>` |
| `TableCell` | Data cell `<td>` |

## Accessibility

- Keyboard navigation for sorting and row actions
- ARIA sort attributes on sortable columns
- Screen reader support for selection state
