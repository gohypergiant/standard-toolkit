# Accordion Group
Groups multiple Accordion components with shared configuration. Use this component when you need related accordions that share variant styling and expansion behavior.

## Example Usage

```tsx
<AccordionGroup>
  <Accordion>
    <AccordionHeader>
      <AccordionTrigger>
        <Icon><Placeholder /></Icon>
        Accordion one
      </AccordionTrigger>
    </AccordionHeader>
    <AccordionPanel>
      <p>First accordion content...</p>
    </AccordionPanel>
  </Accordion>
  <Accordion>
    <AccordionHeader>
      <AccordionTrigger>
        <Icon><Placeholder /></Icon>
        Accordion two
      </AccordionTrigger>
    </AccordionHeader>
    <AccordionPanel>
      <p>Second accordion content...</p>
    </AccordionPanel>
  </Accordion>
</AccordionGroup>
```

## Variants
* **cozy** - Standard size with larger text and spacing (default)
* **compact** - Smaller size with reduced text and spacing for space-constrained layouts

## Props
Extends all [`DisclosureGroup` props](https://react-spectrum.adobe.com/react-aria/DisclosureGroup.html#disclosuregroup-1) from react-aria-components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'cozy' \| 'compact'` | `'cozy'` | Visual style for all child accordions |
| `isDisabled` | `boolean` | `false` | Disables all child accordions |
| `allowsMultipleExpanded` | `boolean` | `false` | Allow multiple accordions to be expanded simultaneously |
| `expandedKeys` | `Iterable<Key>` | - | Controlled set of expanded accordion keys |
| `defaultExpandedKeys` | `Iterable<Key>` | - | Default expanded keys (uncontrolled) |
| `onExpandedChange` | `(keys: Set<Key>) => void` | - | Called when expanded keys change |
| `className` | `string \| ((state) => string)` | - | CSS class name, supports render props |
| `children` | `ReactNode` | - | Accordion components |

See [Accordion](?path=/docs/components-accordion--playground) for individual accordion props.

## Implementation Notes
- Wraps `DisclosureGroup` from react-aria-components
- Provides variant context to all child accordions
- Supports keyboard navigation between accordions
