# Checkbox Group
Groups multiple checkboxes with shared state and configuration. Supports vertical and horizontal layouts with automatic spacing.

## Example Usage

```tsx
<CheckboxGroup label="Notification preferences">
  <Checkbox value="email">Email notifications</Checkbox>
  <Checkbox value="sms">SMS notifications</Checkbox>
  <Checkbox value="push">Push notifications</Checkbox>
</CheckboxGroup>
```
<Canvas of={CheckboxGroupStories.Default} />

### Horizontal Layout
```tsx
<CheckboxGroup label="Features" orientation="horizontal">
  <Checkbox value="feature1">Feature 1</Checkbox>
  <Checkbox value="feature2">Feature 2</Checkbox>
  <Checkbox value="feature3">Feature 3</Checkbox>
</CheckboxGroup>
```

### Controlled
```tsx
const [values, setValues] = useState(['email']);

<CheckboxGroup label="Preferences" value={values} onChange={setValues}>
  <Checkbox value="email">Email</Checkbox>
  <Checkbox value="sms">SMS</Checkbox>
</CheckboxGroup>
```

## Props
Extends all [`CheckboxGroup` props](https://react-spectrum.adobe.com/react-aria/CheckboxGroup.html#props) from react-aria-components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | - | Group label displayed above checkboxes |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Layout direction |
| `value` | `string[]` | - | Controlled selected values |
| `defaultValue` | `string[]` | `[]` | Default selected values |
| `onChange` | `(value: string[]) => void` | - | Called when selection changes |
| `isDisabled` | `boolean` | `false` | Disables all checkboxes in group |
| `isRequired` | `boolean` | `false` | Whether selection is required |
| `classNames` | `object` | - | Custom class names (see below) |

**classNames:**
| Key | Description |
|-----|-------------|
| `group` | Applied to the group container |
| `label` | Applied to the group label |

## Implementation Notes
- Wraps `CheckboxGroup` from react-aria-components
- Horizontal orientation wraps when space is constrained
- Label component shows required indicator when `isRequired` is true
