# Flashcard
A quick visual identifier for displaying multiple data points about a map entity. Shows header, subheader, additional data, and a details list with loading state support.

Components:
- **Flashcard** - Container with loading state and header/subheader props
- **FlashcardHero** - Upper section displaying header and subheader
- **FlashcardAdditionalData** - Secondary data below the hero
- **FlashcardDetailsList** - Wrapper for key-value pairs
- **FlashcardDetailsLabel** - Label for detail items
- **FlashcardDetailsValue** - Value for detail items

## Example Usage

```tsx
const details = [
  { label: 'Status', value: 'Active', id: '1' },
  { label: 'Type', value: 'Aircraft', id: '2' },
  { label: 'Speed', value: '450 kts', id: '3' },
];

<Flashcard isLoading={isLoading} header="Identifier" subheader="DATA">
  <FlashcardHero />
  <FlashcardAdditionalData>SECONDARY_DATA</FlashcardAdditionalData>
  <FlashcardDetailsList>
    {details.map((detail) => (
      <Fragment key={detail.id}>
        <FlashcardDetailsLabel>{detail.label}</FlashcardDetailsLabel>
        <FlashcardDetailsValue>{detail.value}</FlashcardDetailsValue>
      </Fragment>
    ))}
  </FlashcardDetailsList>
</Flashcard>
```

## Props

### Flashcard
Extends all standard HTML `<div>` attributes.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `header` | `string` | - | Primary heading text (required) |
| `subheader` | `string` | - | Secondary text below header (required) |
| `isLoading` | `boolean` | `false` | Shows skeleton placeholders when loading |
| `className` | `string` | - | Custom CSS class |

### FlashcardHero
Displays the header and subheader from Flashcard context. Shows skeleton placeholders when loading.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Custom CSS class |

### FlashcardAdditionalData
Secondary data section. Hidden when loading.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Additional data content |
| `className` | `string` | - | Custom CSS class |

### FlashcardDetailsList
Wrapper around DetailsList with `align="justify"`.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | FlashcardDetailsLabel and FlashcardDetailsValue pairs |
| `className` | `string` | - | Custom CSS class |

### FlashcardDetailsLabel
Label for detail items. Shows skeleton when loading.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Label content |
| `className` | `string` | - | Custom CSS class |

### FlashcardDetailsValue
Value for detail items. Shows skeleton when loading.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Value content |
| `className` | `string` | - | Custom CSS class |

## Implementation Notes
- Uses FlashcardContext to share `isLoading`, `header`, and `subheader` state
- FlashcardDetailsList wraps the DetailsList component
- All detail components show Skeleton placeholders during loading state
