# Flashcard
A quick visual identifier for multiple points of data for a given map entity.

## Example Usage

### Basic

```tsx
  const details = [
    { label: 'key', value: 'value', id: unique-id-1 },
    { label: 'key', value: 'value', id: unique-id-2 },
    { label: 'key', value: 'value', id: unique-id-3 },
    { label: 'key', value: 'value', id: unique-id-4 },
    { label: 'key', value: 'value', id: unique-id-5 },
    { label: 'key', value: 'value', id: unique-id-6 },
    { label: 'key', value: 'value', id: unique-id-7 },
  ];

  <Flashcard>
    <FlashcardHero>
      <FlashcardIdentifier>
        <FlashcardHeader>Identifier</FlashcardHeader>
        <FlashcardSubheader>DATA</FlashcardSubheader>
      </FlashcardIdentifier>
    </FlashcardHero>
    <FlashcardSecondary>
      <FlashcardSecondaryData>SECONDARY_DATA_01</FlashcardSecondaryData>
      <FlashcardSecondaryData>SECONDARY_DATA_02</FlashcardSecondaryData>
      <FlashcardDetailsContainer details={details}/>
    </FlashcardSecondary>
  </Flashcard>
```

## Props
The Flashcard component accepts all standard HTML div attributes.

* **`isLoading`** - Set visual state for loading on the Flashcard component.

### FlashcardHero
Upper half of the Flashcard to provide identifier data for the object being observed.

### FlashcardIdentifier
Header for upper half of flashcard. Container for **`header`** and **`subHeader`**

### FlashcardHeader
Main identifier for Flashcard.

### FlashcardSubheader
Subheading for main identifier.

### FlashcardSecondaryContainer
Container for the lower half of the Flashcard.

### FlashcardSecondaryData
Secondary data information that will appear below the initial header.

### FlashcardDetailsContainer
* **`details`** - Array of additional details of the FlashcardMetaData type.
  * `{ label: string, value: string | number | null, id: UniqueId }` - Additional detail, only first 5 in the array are displayed.
* **`classNames`** - Object for various class names for the details component.
  * `container` - Styling for container for the additional detail list.
  * `item` - Styling for individual entry in detail list.
  * `label` - Styling for detail label
  * `value` - Styling for detail value
