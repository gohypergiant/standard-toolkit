# Label
A semantic label component for form elements with automatic optional/required indicators. Built on React Aria Components.

Components:
- **Label** - Main label component with required/optional text handling
- **LabelProvider** - Context provider for consistent props across multiple labels

## Example Usage

```tsx
<Label>Email Address</Label>

<Label isRequired>Password</Label>

<Label isDisabled>Account Number</Label>
```

## Props
Extends [`Label`](https://react-spectrum.adobe.com/react-aria/TextField.html#label) props from react-aria-components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isRequired` | `boolean` | `false` | Required field (hides "(optional)" text) |
| `isDisabled` | `boolean` | `false` | Disabled styling |
| `children` | `ReactNode` | - | Label text content |
| `className` | `string` | - | Custom CSS class |

### LabelProvider
Context provider for setting default props across multiple Label components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isRequired` | `boolean` | - | Required state for all child Labels |
| `isDisabled` | `boolean` | - | Disabled state for all child Labels |
| `children` | `ReactNode` | - | Label components |

```tsx
<LabelProvider isRequired>
  <Label>First Name</Label>
  <Label>Last Name</Label>
  <Label>Email</Label>
</LabelProvider>
```

## Accessibility
- Renders as semantic `<label>` element
- Works with `htmlFor` attribute to associate with form controls
- Automatic "(optional)" text for non-required fields
- Screen reader support via React Aria

## Implementation Notes
- Wraps `Label` from react-aria-components
- Automatically appends "(optional)" when `isRequired` is false
- Uses data attributes for styling states
