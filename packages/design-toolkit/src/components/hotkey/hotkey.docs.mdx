# Hotkey
Visual representation of keyboard keys and shortcuts. Used to show keyboard shortcuts in documentation, help systems, and UI elements.

Components:
- **Hotkey** - Individual key display (renders as `<kbd>`)
- **HotkeySet** - Container for grouping multiple keys
- **HotkeyProvider** - Context provider for default props

## Example Usage

```tsx
<Hotkey>CMD</Hotkey>
```
<Canvas of={HotkeyStories.Default} />

## Variants

| Variant | Description |
|---------|-------------|
| `outline` | Renders with borders (default) |
| `flat` | Plain text, appropriate for use in MenuItem components |
| `icon` | For icon content with proper sizing, never has a border |

## Props

### Hotkey
Extends [Keyboard](https://react-spectrum.adobe.com/react-aria/Keyboard.html) from react-aria-components.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'outline' \| 'flat' \| 'icon'` | `'outline'` | Visual style |
| `className` | `string` | - | Custom CSS class |
| `children` | `ReactNode` | - | Key label or icon |

### HotkeySet
Container for grouping multiple Hotkey components. Extends all standard HTML `<div>` attributes.

```tsx
<HotkeySet>
  <Hotkey>CMD</Hotkey>
  <span>+</span>
  <Hotkey variant="icon">
    <Icon><MouseRightClick /></Icon>
  </Hotkey>
</HotkeySet>
```
<Canvas of={HotkeyStories.KeysSet} />

### HotkeyProvider
Context provider for setting default props across multiple Hotkey components.

```tsx
<HotkeyProvider variant="flat">
  <Hotkey>Ctrl</Hotkey>
  <Hotkey>C</Hotkey>
</HotkeyProvider>
```

## Accessibility
- Uses semantic `<kbd>` HTML element for keyboard input
- Screen readers announce the content as keyboard keys
- Wraps react-aria's Keyboard component for proper semantics

## Implementation Notes
- Wraps `Keyboard` from react-aria-components
- HotkeySet automatically sizes icons via IconProvider
